# Пресеты настроек

Менеджер **Fallout-MaW: Пресеты настроек** переносит только явно помеченные настройки мира. Настройки клиента и пользователя, состояние миграций, runtime-данные интерфейса и служебные состояния мира в пресеты не входят.

Основной пресет имеет постоянный ID `fallout-maw`. Его можно менять и переименовывать, но нельзя удалить. Новый пресет всегда создаётся как полная независимая копия основного. Выбранный пресет хранится отдельно в каждом мире, а любое последующее изменение управляемой настройки автоматически создаёт его новую полную ревизию.

Системные JSON находятся в `storage/settings-presets/<preset-id>.json`. Использованные файлы дополнительно копируются в `worlds/<world-id>/settings-presets`, поэтому уникальные пресеты мастера восстанавливаются после полной замены папки системы. При совпадении ID системный файл считается авторитетным; пресеты с другими ID не затрагиваются.

Зарезервированный скрытый файл `fallout-maw-migration-seed.json` хранит только исходные defaults для одноразового преобразования старых миров. Он не появляется в менеджере, не может быть выбран или импортирован и после миграции мира больше не участвует в его обычной работе.

## Первый выпуск с пресетами

Перед передачей обновлённой папки системы один раз откройте исходный рабочий мир основным GM. Пока главный файл помечен как первичный seed, система снимет его фактические настройки, сохранит отдельную личную копию с названием мира и финализирует `Fallout-MaW`. Новый пустой мир активирует исходный `Fallout-MaW`, но сам по себе первичный seed не финализирует.

Импорт одного JSON с существующим ID полностью заменяет этот пресет. Новый ID добавляет новый пресет. Удаление обычного пресета действует только в текущем мире и сразу убирает его из локальной библиотеки; оно не передаётся другим мастерам. Старый экспорт baseline допускается только при импорте и преобразуется в новый пресет с новым ID.

Публичный API доступен как `CONFIG.FalloutMaW.settingsPresets`: `list`, `get`, `active`, `create`, `activate`, `rename`, `remove`, `import`, `export`, `refresh` и `flush`.
